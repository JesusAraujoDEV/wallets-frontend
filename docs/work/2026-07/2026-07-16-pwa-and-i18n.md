# 2026-07-16 — PWA (Phase 1) + language selector (ES/EN/DE)

## What changed
Implemented Phase 1 of `docs/proposals/mobile-app-plan.md`: the app is now installable (manifest, generated icon set, service worker via `vite-plugin-pwa`, iOS meta tags). Added a persisted language preference (Spanish/English/German) via `react-i18next`, with a `LanguageSwitcher` in the sidebar and translation coverage for the navigation shell, onboarding tour, Rates screen, and transaction filters. Also moved the backend's default port from 3001 to 4001 (`.env`, `.env.example`, `index.js`) and updated every frontend `VITE_BACKEND_URL` reference to match.

## Why
The owner read the mobile app plan (committed earlier today), which recommends PWA as the low-risk first phase, and asked to implement it now rather than leave it as a future-only proposal. Also asked for a language selector "in the best possible way" and for the backend off port 3001 (which has repeatedly collided with an unrelated local process this session).

## How
- **PWA**: `vite-plugin-pwa` with `registerType: "autoUpdate"`. Icons generated with Pillow (no existing brand icon asset at square/multi-size — `public/platica_background.png` is a 1024×559 hero image, not a logo) as a mint-green circle "P" monogram matching the sidebar wordmark's accent color, at 192/512/512-maskable/180 (apple-touch-icon) sizes. `workbox.navigateFallbackDenylist` excludes `/api/*` so API calls are never served from the service worker cache — financial data is always live. `maximumFileSizeToCacheInBytes` raised to 5MB because the app ships a single ~2.4MB unsplit JS bundle (flagged separately in the frontend audit as a bundle-splitting opportunity, not fixed here — out of scope for "make it installable").
- **i18n**: `react-i18next` + `i18next-browser-languagedetector`, `fallbackLng: "es"` (the app's existing default), detection order `localStorage` → `navigator`, persisted under `pwi_language`. Three flat-namespace resource files (`src/locales/{es,en,de}.json`) covering `common`, `nav`, `sidebar`, `onboarding`, `rates`, `filters`, `dashboard`. Wired into the sidebar nav, `DesktopSidebar`/`MobileSidebarHeader`, `OnboardingTour`, `Rates` page + its three sub-components, and `TransactionFilters`.
- **Port**: `4001` chosen arbitrarily to avoid the specific process that kept colliding with 3001 earlier in the session; no significance beyond that.

Verified in-browser: production build (`npm run build` + `vite preview`) confirms manifest served, service worker registered, all four icon files return `200 image/png`. Language switch confirmed live (Spanish → English → German → back to Spanish) on the sidebar and the Rates page, including persistence across a reload.
